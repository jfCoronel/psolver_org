History

Reverse time evolution (from bottom to top)

Only changes since the first English version (version 4.0) are shown.

V 4.49 (13 January 2026)

Edition of solution and statement from the task

  • Two buttons are added to be able to edit the statement and the solution directly from the task management.
  • An error that generated multiple dialog boxes when the task is sent to the students is corrected.
  • An error is corrected when switching between the student tables and student variable tables when some column was sorted.

V 4.48 (9 January 2026)

Save button

  • A save button is added to the button bar to save changes made to the problem without having to exit it. The button will appear in red when there are unsaved changes.
  • The background color of components with errors is now stronger to make it more visible.
  • It is checked that the user does not leave the text component empty, showing an error message if they do.
  • A memory save of the problem is made if the page is exited, the back button is pressed or reloaded or the tab is closed, when re-entering it is automatically restored.

V 4.47 (9 January 2026)

Improved task correction

  • In task correction, two different tables can be viewed: one by student and another by variable. This second table allows you to quickly see which variables have been answered correctly by all students and which have not.
  • Reordering of the buttons in the task correction table to improve usability.
  • The correction tables in the task editor only apply to the students of the course that has that task selected, allowing the same task to be changed from course without the need to create another new task.
  • A maintenance system is added to be able to delete old problems from inactive users from the Firebase database

V 4.46 (10 December 2025)

New problem copy management

  • New policy for copying or duplicating problems: Activities cannot be copied, statements and solutions can only be copied by teachers and normal problems depending on whether they are private or not.
  • In the self-correction of activities, the solution ID is not visible, only the active or inactive correct button appears.
  • Only own problems can be imported from a file, not those of another user.
  • Editing / Updating the user name directly in the user information that appears in the problem list.
  • The creation date of the user information is removed and the number of problems is changed to the format n of n_max
  • Error in unit change written as " x = x/10 " did not work, nor gave an error, now it works.

V 4.45 (1 December 2025)

Course management with table

  • Course management is changed to a table, with an update button and a message showing the date of the last update.
  • The format of the buttons in the task editing is changed to small buttons.
  • Waiting message on the buttons of the course and task editors while the action is being performed.
  • The name of the problem becomes an editable field when clicked or hovered over.
  • An error is corrected where double-clicking on a text did not focus the Quick Editor.

V 4.44 (1 November 2025)

Submit activity with button

  • The link "Submit activity" is changed to a button.
  • The label and the submit button are placed to the right of the name, leaving the description on the lower line.
  • The label that appeared on the paper showing the submission date of the activity is removed.
  • An error is corrected in which equations were not rendered when only texts were present.

V 4.43 (24 October 2025)

User configuration

  • User settings are saved in the database and edited collectively.
  • A “Solving” message is added for problems that take more than 2 seconds to solve.
  • The name of the problem is changed to editable field.
  • The display of sections when printing or generating PDFs is removed.
  • An error with the creation date that produced a gray screen is corrected.
  • An error that occurred with the names of fluids defined in global sections is corrected.
  • An error that did not detect the duplication of symbols with global sections is corrected.

V 4.421 (19 September 2025)

Error in automatic table recalculation

  • It corrects an error in the automatic recalculation of tables that caused them not to be recalculated correctly.

V 4.42 (8 September 2025)

Fluid property diagrams

  • It has been added the possibility to create fluid property diagrams in the graphical components. In a new section called "Fluid Property Diagrams" you can create different types of diagrams: for pure fluids (T-p, h-log p, T-s) and for humid air the psychrometric diagram.

V 4.42 (8 September 2025)

Fluid property diagrams

  • It has been added the possibility to create fluid property diagrams in the graphical components. In a new section called "Fluid Property Diagrams" you can create different types of diagrams: for pure fluids (T-p, h-log p, T-s) and for humid air the psychrometric diagram.

V 4.41 (10 July 2025)

Creating new variables in tables

  • It is now possible to create new variables in tables and indicate their value for each of the rows of the table, using expressions separated by commas. This option allows you to create tables and graphs with variables that are within the same problem without having to solve the problem again.
  • In the definition of the data, reference can be made to previously created questions. If the question has a numerical value, that value will be used, if it is a list of options, a field has been added to this type of questions to specify the numerical value for each option.
  • Questions inserted in text boxes are made to lose focus when clicked elsewhere.

V 4.40 (2 may 2025)

New component Question

  • A version is implemented to be able to undo the last actions, although the operation is not perfect, it allows to undo many of the modifications to the problems using CTRL/CMD + Z.

V 4.35 (24 april 2025)

CTRL/CMD + Z Implementation

  • A version is implemented to be able to undo the last actions, although the operation is not perfect, it allows to undo many of the modifications to the problems using CTRL/CMD + Z.

V 4.34 (7 april 2025)

Shortcut to calculate all sections (R)

  • Added a keyboard shortcut to calculate all sections of a problem by pressing the R key.
  • Changed the symbol to delimit variables in texts from ## to ??, due to incompatibility with Markdown titles.
  • Simplified New... button

V 4.33 (3 april 2025)

Valores del problema en los textos

  • Added the ability to automatically include values (data, unknowns, fluids or wet air) in Markdown texts by placing the variable symbol between ##. For example: ## x_1 ## would put in the text the value of that variable followed by its unit.
  • The value of the data entered in the tables will always be the value without changing the unit. When solving the problem, the unit change will be made if it exists.
  • An error is corrected in which the lists of the activities of a task also included the old activities that used the same statement as the task.

V 4.32 (3 February 2025)

Automatic recalculation of tables

  • A check is added to the tables to allow automatic recalculation of the tables when solving the problem.
  • Fixed a bug in writing the csv file for activities.
  • Other minor changes.

V 4.31 (20 January 2025)

Improvements in courses and tasks

  • Sorting and filtering of the course list and task list.
  • Duplication of courses and tasks.
  • Saving table of corrected activities of a task.
  • When sending activities to all students the system checks that it does not exist, so as not to duplicate it.
  • It is possible to import external courses to the user (from another teacher) through the course id.
  • A button has been added to activate the self-correction of the activities from the task, by pressing it we allow the student to self-correct his problem.
  • Added “Hide unit change” check for data and unknowns.
  • Print button in the problem button bar.

V 4.30 (21 October 2024)

Courses and tasks

  • New management of activities through courses and tasks that teachers can create.
  • The new activity button is changed to the management of activities and courses where the user can register in a course or create a new activity.
  • Change in the visualization of the tables, differentiating the tables that go on the “paper” and the tables of the interface.
  • Change in the toolbar, configuration on the left and the user on the right with an icon and a tooltip (to avoid variable width).
  • The type “Solution” is added as one more type of problems.
  • The selection of the type of problem is done by a drop-down list in the general properties of the problem.
  • The paper height is adjusted for A4.

V 4.26 (12 September 2024)

Text type data correction

  • The correction of text type data, which are those whose value is written between quotation marks, is allowed.
  • Some minor errors in the list of activities associated with a statement have been corrected.

V 4.25 (23 April 2024)

Server changed to Github

  • The application server is moved to a Github page for speed reasons. https://jfcoronel.github.io/psolver_org/app
  • Added the name of the author (user) in the list of activities associated with a statement.
  • The click area for editing the general properties of a problem is expanded to the whole upper area, previously only the problem name was used.
  • Fixed some minor bugs.

V 4.24 (11 April 2024)

Component Lockout

  • The possibility of locking individual components by means of a check box has been added, thus preventing them from being modified or deleted unintentionally.
  • Fixed some minor bugs./li>

V 4.23 (18 March 2024)

A3 page format

  • A3 page size (portrait/landscape) is added to the properties of each of the problems.
  • The upper width limit on images and graphics is changed so that it can be greater than 100% on A3 or landscape pages.
  • Fixed some minor bugs./li>

V 4.22 (29 February 2024)

Landscape page format

  • Page orientation (portrait/landscape) is added to the properties of each of the problems.
  • The page setup of the problem list is stored so that it does not change when returning to the list from the problems.
  • Fixed some minor bugs.

V 4.21 (24 January 2024)

Errors in copying statements

  • Standard users (students) are not allowed to copy statements, only teachers are allowed to do so.
  • Problems that are statements are removed from the list of activities linked to a statement. Preventing the statement itself and any copy of it from appearing.

V 4.20 (17 January 2024)

Types of users: Students and Teachers.

  • Two different types of users are established for the application: Standard (student) and Teacher. The difference is that only Teacher type users will be able to create "Statements".
  • The value of the maximum number of problems is changed to 50 for standard users and 250 for teachers.
  • El número máximo de problemas se muestra en la información general del usuario.
  • The maximum number of problems is shown in the user's general information.
  • The creation of a new activity is done via a separate button from the copy problem button.
  • Teachers can edit student activities, even if they are already submitted
  • Links to images in Google Drive are changed, due to a problem with direct links from google drive that was detected as of 11 January 2014 (https://issuetracker.google.com/issues/319531488?pli=1).

V 4.13 (5 December 2023)

Redefining, simplifying and reordering

  • Rearrangement of the button bar, left back arrow, separation between edit and resolution buttons.
  • Clicking on IDs copies their value to the clipboard.
  • Problem types are redefined: "Regular", "Statement", "Activity". To create a statement (previously called "Teacher activity") a regular problem will be created and in its properties the "Statement" check is used. It is therefore possible to switch between regular problem and statement.
  • "Activities" formerly called "Student Activity" are created by copying a statement.
  • In the statements the id of the correct problem or solution problem is stored.
  • In the list of activities linked to a statement, some errors are corrected and the export to CSV and the personalised configuration of the pagination are added. In addition, the number of correct, incorrect and not found are separated into columns.
  • he quick messages disappear all by time, some of them stayed when an error occurred.
  • In the unknowns the unit change is shown even if the unknown is not yet calculated.

V 4.12 (30 October 2023)

Correction of problems with tolerance

  • In correction by comparison of problems, the values are considered correct when the error is less than 0.1%. In case the correct value is 0, the user value must be less than 1e-9. It does not perform correction of the string used for the unit.
  • When trying to solve a problem where one of its active components has an error, the program will display an informational message and suspend the resolution.

V 4.11 (19 September 2023)

Improving quick editing

  • The responsiveness of quick editing (double-click) of data, unknowns and equations is improved.
  • A link to the "Documentation" is added to the top area of the registration screen and the problem list.

V 4.10 (11 September 2023)

Improving section management

  • Global sections: the components of these sections will be used in all other sections of the problem.
  • Correction by comparison can already be used in problems with sections.
  • Possibility to make problems "private", private problems cannot be viewed or copied.
  • New filter by type of problem: "normal", "activity as teacher", "activity as student".
  • The language change can only be made on the initial registration screen.
  • Fixed error in solving problems with data equal to "?".
  • Error correction in the tables placed in sections.
  • Fixed error in some messages after translation into English.

V 4.00 (12 July 2023)

Name change, we are now pSolver

  • Change of name and logo. New name pSolver, new website http://psolver.org
  • First English version of the application. The user can choose between the English or the Spanish version.
  • Multiple components are selected by pressing ctrl/cmd and dragging the mouse. A box with a dashed border will be shown.
  • In the text we can insert equations in the same format used by the application, surrounding them with $$.
  • It is now possible to use incompressible fluid names in string type data. http://www.coolprop.org/fluid_properties/Incompressibles.html
  • The gamma function (Euler's Gamma, a generalisation of the factorial) is removed, in order to be able to use gamma as a variable.
  • The tabular view can be sorted by sections.
  • You can click inside the quick edit boxes to change the cursor position.
